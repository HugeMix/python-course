Урок 13: Основы ООП (Классы и объекты)

Познакомимся с объектно-ориентированным программированием в Python.

🎯 Цель урока

Научиться создавать классы и объекты, работать с атрибутами и методами, понять принцип инкапсуляции.

📌 Что такое ООП?

Объектно-ориентированное программирование (ООП) - это стиль программирования, где код организован в виде объектов, которые объединяют данные и действия над ними. ООП помогает структурировать большие программы и делать их удобными для поддержки.

🔧 Класс - чертёж для объектов

Класс - это шаблон (чертёж), по которому создаются объекты. Давайте создадим класс Car:

class Car:
    def __init__(self, brand, model, year):
        self.brand = brand  # Атрибут марки
        self.model = model  # Атрибут модели
        self.year = year    # Атрибут года выпуска

    def show_info(self):
        print(f"Автомобиль: {self.brand} {self.model}, {self.year} года выпуска")

Метод __init__ - это специальный конструктор, который вызывается автоматически при создании нового объекта. В нём задаются начальные атрибуты.

🚗 Объект - экземпляр класса

Объект - это конкретный экземпляр, созданный по шаблону класса. Один класс - много объектов:

my_car = Car("Toyota", "Camry", 2020)
my_car.show_info()
# Выведет: Автомобиль: Toyota Camry, 2020 года выпуска

⚙️ Методы класса

Методы - это функции, определённые внутри класса. Они описывают поведение объекта. Добавим метод start_engine:

class Car:
    def __init__(self, brand, model, year):
        self.brand = brand
        self.model = model
        self.year = year

    def show_info(self):
        print(f"Автомобиль: {self.brand} {self.model}, {self.year} года выпуска")

    def start_engine(self):
        print(f"{self.brand} {self.model}: Двигатель запущен!")

my_car = Car("BMW", "X5", 2022)
my_car.show_info()
my_car.start_engine()

👤 Параметр self

Каждый метод класса обязательно принимает первым параметром self - это ссылка на конкретный объект. Благодаря self метод знает, с какими данными работать. Без него Python не сможет обратиться к атрибутам объекта внутри метода.

💡 Принцип инкапсуляции

Инкапсуляция - один из ключевых принципов ООП. Это умение объекта скрывать свои внутренние данные от внешнего кода ради безопасности и защиты от случайных изменений.

В Python «приватность» обозначают двойным подчёркиванием перед именем атрибута:

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # Приватный атрибут (нельзя изменить снаружи напрямую)

    def get_balance(self):
        return self.__balance     # Безопасный доступ через метод
